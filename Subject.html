Bases du Ruby

Les variables

En programmation, une variable correspond à un emplacement de stockage contenant une information. L’information peut être sous forme de nombre entier, nombre à virgule, phrases…

Lorsque l’on crée une variable on parle de déclaration de variable ;

Pour parler de l’information stockée dans une variable, on nomme l’information valeur de la variable.

Lorsque l’on modifie la valeur d’une variable on parle d'assignation

Voici un exemple en Ruby:

charactere = 'b'
une_variable = "je suis une variable"
chaine = "coucou je suis un pokemon"
nouvelle_variable = une_variable
nb = 3
nb_number_2 = 49.5

Vous pourrez noter quelques points:

Pour assigner un caractère à une variable il faut utiliser les simples quotes ' tel que : caractere = 'b' ;

Pour assigner une chaîne de caractère à une variable il faut utiliser les " tel que : chaine = "je suis un pokemon" ;

Il possible d’assigner la valeur d’une variable à une nouvelle variable : nouvelle_variable = une variable ;

Un nom de variable ne peut prendre de caractère espace ' ', dans son nom ;

Fonction

Definition

Une fonction est un ensemble d’instructions visant à réaliser une tâche en particulier. Une fonction peut être appelée par une autre afin d’effectuer la tâche que l’on attend d’elle.

On considère qu’une fonction peut :

  • en appeler une autre ;
  • être appelée par une autre fonction ;
  • s’appeler elle même – c’est la notion de récursivité ;
  • prendre une ou plusieurs valeurs en guise de paramètre ;
  • renvoyer une valeur en guise de résultat.

Un paramètre de fonction est une valeur qui lui est envoyée au moment où elle se fait appeler. Il s’utilise dans le corps de la fonction comme une variable. D’autres part, il est possible qu’une fonction ne prenne pas de paramètre.

Exemple

La fonction put(variable), prend une variable en argument et affiche la valeur de la variable.

Voici un exemple plus complet:

variable = "Je suis une variable nommée variable avec toute cette phrase en valeur"
puts(variable)

Exemple 2

Nous allons à présent regarder comment déclarer / créer une fonction nous même:

def function(var1, var2) 
	puts(var1)
	puts(var2)
	return var1
end

Explication

  • def function(var1, var2), le mot clef def permet de signaler le début d’une fonction, le mot clef est suivi par le nom de la fonction (ici nommée function) et par les paramètres entre parenthèses (var1 et var2).
  • return var1, permet de renvoyer la valeur de var1 à l’appelant.
  • end, ce mot permet de signaler la fin d’utilisation d’un mot clef ici def

Exemple 3 : Retour d’une fonction

Voici un exemple de l’utilisation d’un retour de fonction:

def ret_func
	var = "je fais une variable"
	return var
end
pikachu = ret_func
puts(pikachu)

Explication

  • def ret_func, vous noterez ici que la fonction ne prend pas d’argument en paramètre ;
  • return var, on retourne la valeur de var à l’appelant ;
  • pikachu = ret_func, on assigne le retour de la fonction à la variable pikachu ;
  • On affiche la valeur de la variable pikachu dans le terminal

Paramètre d’un programme

Lors de l’exécution d’un programme il est possible de récupérer les paramètres passés en argument tel que:

ruby prog.rb "Argument numéro 0" "Argument numéro 1"

Ici nous lançons notre programme avec deux arguments “Argument numéro 0” et “Argument numéro 1”. Pour les récupérer dans notre programme il faut utiliser un tableau nommé ARGV, ce tableau s’utilise de la manière suivante:

puts(ARGV[0])
puts(ARGV[1])

  • ARGV[0] contient “Argument numéro 0”

  • ARGV[1] contient “Argument numéro 1”

  • Vous noterez que le premier argument est stockée à l’emplacement 0 : ARGV[0] et non à 1.

Copiez et exécutez le code ci dessus pour comprendre !

Si, sinon si, sinon

La forme la plus commune de test est le test si. En ruby, il est noté if. Il a pour rôle de vérifier qu’une condition est vraie pour exécuter un bloc de code.

Il est possible de joindre à la suite d’un if une seconde condition elsif – littéralement : sinon si – pour exécuter un bloc lorsque la condition du if précédent n’est pas validé et que la condition du elsif est vraie .

De même il est possible de joindre à la suite d’un if ou elsif un else – littéralement : sinon – pour exécuter un bloc de code lorsque la condition du if précédent n’est pas validée.

Les opérateurs de comparaison

Nous avons tout à l’heure fait la connaissance d’opérateurs qui permettaient de faire des opérations mathématiques. Les opérateurs que nous allons découvrir ici ne servent pas à faire des maths mais à comparer deux éléments, vous allez vite comprendre.

  • == permet de vérifier une égalité ;
  • != permet de vérifier une différence ;
  • >, >=, < et <= permettent de vérifier si une valeur est supérieure, supérieure-ou-égale, inférieure ou inférieure-ou-égale à une autre.

Exemple

var = "pokemon"
if var == "pokemon"
	puts("La variable est un pokémon")
end

var = "pokemon"
if var == "pokemon"
	puts("La variable est un pokémon")
else
	puts("La variable n'est pas un pokémon")
end

var = "pichu"
if var == "raichu"
	puts("var est un raichu")
end
if var == "pikachu"
	puts("var est un pikachu")
end
if var == "pichu"
	puts("var est un pichu")
end

var = 3
if (var % 2) == 0
	puts("La variable est pair")
else
	puts("La variable est impair")
end

var = "juillet"
if var == "aout"
	puts("C'est bientôt la rentrée")
elsif var == "septembre"
	puts("C'est la rentrée !")
else
	puts("vacances !")
end
  • end, ce mot permet de signaler la fin d’utilisation d’un mot clef ici if

Ruby2d

Définition : Une librairie graphique

Une librairie en programmation est un ensemble d’outils, de fonctions utilisables dans un contexte et un but précis. En effet, aujourd’hui nous allons utiliser une librairie qui a pour but de déssiner des objets, images dans un fenêtre, mais il existe aussi des librairies mathématiques (fonctions: cos, sin, puissances…), ou audio. Chaque langage dispose d’une multitude de librairies disponibles.

Une fenêtre

Pour commencer créons un nouveau fichier avec emacs, nommons-le window.rb.

En Ruby, pour utiliser une librairie il faut utiliser le mot clef require [NOM DE LA LIBRARIE], puisque nous utilisons la librairie ruby2d nous allons écrire: require 'ruby2d' au début de notre code.

Voici quelques fonctions utiles en ruby2d:

fonction usage
set width: [taille] modifie la largeur de la fenêtre
set height: [taille] modifie la hauteur de la fenêtre
set title: “[nom de la fenêtre]” modifie le nom de la fenêtre
show affiche une fenêtre basique

Un exemple plus concret :

Boucle d’exécution

Actuellement notre programme est statique, la fenêtre reste vide sans intéraction possible, pour modifier cela, il faut créer une boucle d’exécution qui permetta de modifier le contenu de la fenêtre dynamiquement.

tick = 0 # compteur pour le nombre d'image affiché à l'écran

update do # update permet de vérifier que la fenetre est encore active
  if tick % 10 == 0 # on vérifie qu'il y a eu 10 images d'affiché avant d'actualiser
    set background: 'random' # on change la couleur de fond d'écran
  end # fin du if
  tick += 1 # on incremente le nombre d'image affiché
end # fin du do

L’arc en ciel devrait se présenter dans votre fenêtre.

Vous remarquerez que les mots clefs do et if doivent finir par un end pour préciser la fin de l’action.

Exemple plus complet:

require 'ruby2d'

set width: 1280
set height: 720
set title: "Coding Club @ Bruxelles"

tick = 0 # compteur pour le nombre d'image affiché à l'écran

update do # update permet de rafraichir la fenêtre
  if tick % 10 == 0 # on vérifie qu'il y a eu 10 images d'affiché avant d'actualiser
    set background: 'random' # on change la couleur de fond d'écran
  end # fin du if
  tick += 1 # on incremente le nombre d'image affiché
end # fin du do
show

Placer une image

img = Image.new(		# Appel au contructeur d'un objet Image
				# et assignation à la variable img
	x: 0, y: 0,		# Set l'origine de l'image
	path: "./Kidoo.png"	# Chemin vers l'image à utiliser
)
img.width = 1280		# Largeur de l'image
img.height = 720		# Hauteur de l'image
img.color = "red"		# Changement de couleur l'image

A Noter : Pour déclarer une image, il n’est pas nécessaire de le faire sur plusieurs lignes vous pouvez simplement faire : img = Image.new(x: 0, y: 0, path: "./ECCSC.jpg")

Avec ce code vous avez les bases pour manipuler des images avec la lib Ruby2D.

La variable path dans le constructeur de l’image doit correspondre au chemin d’une image valide ! N’hésitez donc pas à en télécharger une et à l’utiliser.

Exemple plus complet avec un déplacement de l’image:

require 'ruby2d'

set width: 1280
set height: 720
set title: "Coding Club @ Bruxelles"

img = Image.new(x: 0, y: 200, path: "./Kidoo.png") # Creation de l'image
tick = 0 # compteur pour le nombre d'image affiché à l'écran

update do # update permet de rafraichir la fenêtre
  if tick % 10 == 0 # on vérifie qu'il y a eu 60 images d'affiché avant d'actualiser
  end # fin du if
  tick += 1 # on incremente le nombre d'image affiché
end # fin du do

show

Les Carrés

Commençons par une forme basique : le carré.

L’exemple ci-dessous vous montre comment créer un carré.

Square.new(x: 100, y:100, size: 10) # Creation d'un carré de 10x10 en position (100;100)

Et voila vous savez placer un carré ! Facile non ? Essayons de rentre ça un peu plus coloré …

require 'ruby2d'

set width: 1280
set height: 720
set title: "Coding Club @ Bruxelles"

square = Square.new	# Creation d'un objet Carré

square.x = 150		# Set de la posion X à 150
square.y = 100		# Set de la posion Y à 100
square.size = 200	# Set de sa taille à 200
square.color = ['red',	# Ici on utilise un tableau de valeures
	'green',	# ['red', 'green', 'blue', 'yellow']
	'blue',		# representant chacun un coté du carré

show

Plutôt sympa non ?

A Noter : Par défaut la couleur d’un carré est blanche et il est possible de changer sa couleur de façon uniforme en tapant par example square.color = 'red' ou en passant au constructeur la couleur Square.new(color: 'red')

Tout à l’heure nous parlions de rectangles, non ? Au final, un carré est un rectangle particulier 🧐

Les Quad

Ici, nous ne parlons pas de véhicule, mais bien d’une forme : un Quad est une forme géométrique à quatre cotés organisé de la façon que l’on veut.

Par exemple:

Quad.new({  
	x1: 0, y1: 0, 
	x2: 100, y2: 0, 
	x3: 100, y3: 100, 
	x4: 0, y4: 100
})

Ici on a construit un carré a l’aide du Quad en plaçant 4 couples de points (x1;y1 - x2; y2 - x3;y3 - x4;y4) mais on peut le déformer facilement !

L’image ci dessous vous permet de comprendre où sont placés les couples de points :

Et avec le code suivant, on obtient un Losange :

require 'ruby2d' 

set width: 1280
set height: 720
set title: "Coding Club @ Bruxelles" 

quad = Quad.new({
	x1: 100, y1: 100,

})

show

A Noter : pour modifier la couleur d’un Quad il faut modifier la valeur de quad.color : quad.color = 'red'

La Souris

mouse_down, mouse_up
on :mouse_down do |e|
  puts "Mouse down key event caught!"
end

on :mouse_up do |e|
  puts "Mouse up key event caught!"
end

Ici on gère deux cas : :mouse_down qui s’active lorsqu’un bouton de la souris est appuyé et :mouse_up qui s’active lorsqu’un bouton de la souris est relâché.

Le e ici correspond à un objet MouseEvent et contient des informations concernant les actions :mouse_down et :mouse_up :

appel valeurs description
e.type :down ou :up le button a été appuyé ou relâché
e.button :left, :right ou :middle Correspond au bouton utilisé
e.x et e.y n/a position en x et en y cliqué sur la fenêtre

Le Clavier

on :key_down do |e|

end

on :key_held do |e|
  puts "#{e.key} is held down!"
end

on :key_up do |e|
  puts "#{e.key} was released!"P
end

Gérer les événements au clavier peut se faire de trois façons différentes :

  • Au moment ou la touche est appuyée avec :key_down
  • Au moment ou elle est relâchée avec :key_up
  • An moment ou elle est maintenue avec :key_held

Dans ce cas le e correspond à un KeyEvent qui ne contient que deux champs :

  • e.type qui peut prendre les valeurs :down, :up et :held
  • e.key qui correspond au nom de la touche utilisée
